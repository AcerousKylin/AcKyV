## 实现过程

#### 说明

作业整体没有采用统一的警务系统风格，更多的是针对要求实现功能。

详细的代码书写和说明在对应窗体的代码中均有注释进行说明。

#### 基类 - BasicForm

在Program.cs中建立窗口基类，用于静态存储当前用户和密码，为后续针对用户操作的功能提供便利。

登录界面和主界面由于其涉及用户操作，均继承自该基类。

#### 登录界面（登录界面）

1. 手绘登录界面，锁定窗口大小。
2. 对用户名进行检查
   - 是否全为大、小写字母；
   - 是否输入错误；
   - 是否为空。
3. 对密码进行检查
   - 是否长度为10；
   - 是否输入错误；
   - 是否为空。
4. 由于对用户名和密码的分别检查过于冗杂，将其两两分组进行报错。
5. 实现登录与退出按钮。

#### 主界面（主界面）

1. 按要求设置了菜单栏、工具栏（及其图标）和状态栏（静态），并实现了相关功能。

   - 可实现的功能有：
     - 菜单栏：系统、辅助->发送消息（通往**公告发送页面**）、窗口->接警窗口（通往**核心页面**）
     - 工具栏：下岗、接警窗口

2. 额外增加更改密码和下岗的功能。

   - 更改密码拥有与登录时输入密码同样的输入检测功能。
   - 下岗功能可以返回登录界面进行登录或退出。

3. 对右下角的反馈单窗口进行了本地化数据的读取功能。

4. 单击窗口中心的空白部分将在鼠标周围显示一个警徽样式的六角形、双击则可以进入**报表窗口**。

   - 六角形绘制过程：

     基本思路：获取鼠标点击时对控件的相对位置，利用计算无误的12个坐标点进行多边形绘制，最后在6个凸角绘制圆形（扇形）。

     建立一个返回二维PointF数组的六角形定点函数，传入鼠标的点击位置和图像的大小（由于几何变换过于复杂，使用坐标修改较为灵活）。

     在控件的点击事件中获取鼠标的相对坐标并设定倍率，传入上述函数获取12个点和6个凸点、6个凹点坐标，根据坐标进行三层多边形和圆点的绘制。

5. 其余控件均为静态。

#### 接警窗口（核心窗口）

1. 对接警信息的各个控件进行了布局与图标设置。
2. 为所有Combo Box设置了不同的、合理的可选择值。
3. 使用沈阳市公安局的分局设置进行调度单位的操作。
   - 分局选择的交互模式：使用鼠标左右键单击按钮的方式进行交互，左键将该分局添加至待调度单位，右键将该分局添加至已调度单位。一个分局无法同时设置待调度和已调度。
   - 删除与调度的交互模式：使用按钮。
     - 删除：选中的单位将被删除。
     - 调度：双向可调度，选中单位后使用箭头按钮即可调度。
4. 其他控件均为静态。

#### 发送消息页面（公告发送页面）

1. 完成了基础的控件布局。
2. 发送单位是通过本地文件的文件结构进行生成。
   - 通过代码生成了静态数据，作为显示参考；
   - 对对象的类型进行区分并添加不同的图标；
   - 双击文本文档对象将弹出包含其中文本的窗体。
3. 发送按钮可以将内容标题+发送内容中的文本发送至左侧选中的文本文件中。
4. 清空内容可以将发送内容中的文本清空。
5. 其他控件均为静态。
